2.What is git

Git is a powerful version control system that allows developers to track changes, collaborate, and manage software projects. Git Bash is a command-line interface (CLI) for using Git on Windows systems. To install Git Bash, download the latest version from the official Git website and follow the on-screen instructions. After installation, you can launch Git Bash from the Start menu or by searching for it on your computer. Git Bash provides a Unix-like environment, allowing you to run Git commands and access a variety of tools and utilities.

3.Installing Git

go to https://git-scm.com/download/win

Once the download is complete, run the Git installer (.exe file) that you downloaded

Setup Wizard: Follow the instructions in the setup wizard. You can generally accept the default settings unless you have specific preferences to customize.

Choose Components: During the installation process, you'll have the option to choose which components to install. Make sure "Git Bash Here" is selected if you want to use Git from the command line.

Adjusting your PATH environment When prompted, choose the default option "Use Git from the Windows Command Prompt" to add Git to your PATH environment, which allows you to use Git from the command line.

Complete Installation: Once the installation is complete, click "Finish" to exit the setup wizard.

4.Configuring Git

1.Open Git Bash and run the git config --global user.name "Your Name" command to set your name for Git commits. 1.Run git config --global user.email "your@email.com" to configure the email address associated with your Git account. 1.Use git config --list to view all your current Git configuration settings and ensure they are correct.

5. Intializing a Git repository

1.Create a New Repository

Run the git init command in your project directory to initialize a new Git repository and start tracking your files. 2

2.Check Repository Status

Use the git status command to view the current state of your repository, including any untracked or modified files. 3

Add Files to Staging

Utilize git add. to selectively add files to the staging area, preparing them for the next commit.

6. Adding files to the staging area

To add files to the Git staging area, use the git add . command followed by the file name or directory. This prepares the changes to be committed to the repository. For example, git add my_file.txt will stage the file "my_file.txt" for the next commit. You can also use wildcards like git add *.txt to stage all text files in the current directory

7. Commiting changes

After adding files to the staging area using git add , the next step is to create a commit. Commits are snapshots of your bbase that you can reference later. To create a commit, use the git commit -m "Commit message" command. The -m flag allows you to provide a short, descriptive message explaining the changes in this commit.

8. Branches

Branching means you diverge from the main line of development and continue to do work without messing with that main line.The way Git branches is incredibly lightweight, making branching operations nearly instantaneous, and switching back and forth between branches generally just as fast. Unlike many other VCSs, Git encourages workflows that branch and merge often, even multiple times in a day. Understanding and mastering this feature gives you a powerful and unique tool and can entirely change the way that you develop.

Creating a branch

Use the git branch [branch-name] command to create a new branch. This allows you to work on a feature or bug fix independently of the main code base.

10. Viewing all branches

Viewing Branches the git branch command lists all branches in your repository. The active branch is marked with an asterisk. You can also use git branch -a to see remote branches.

11. Switching active branches

Checkout a Branch

Use the git checkout [branch-name] command to switch to a different branch. This allows you to work on a specific feature or bug fix separately from the main codebase.

Create and Switch Combine

git checkout -b [new-branch-name] to create a new branch and immediately switch to it in one command.

12. Merging a branch

Merge a Branch To integrate the changes from a branch into the main bbase, use the git merge [branch-name] command. This will combine the specified branch with the currently active branch.

13. Deleting a branch

Delete a Branch Once a branch has been merged, you can safely delete it using the git branch -d [branch-name] command. This will remove the branch from your local repository.
